# План внедрения телеметрии

## Цель
Реализовать веб-приложение для демонстрации ВКР: запись и отображение маршрутов транспорта на карте Яндекс.Карт из двух MVP-источников (браузерная геолокация и симулятор), с возможностью показать как онлайн-движение, так и уже записанный маршрут.

## Область работ
- Доменная модель и слой персистентности для точек телеметрии.
- Бизнес-сервис для записи и чтения истории телеметрии.
- API-слой для внешнего приема данных и клиентского чтения.
- Миграции данных и стратегия индексации.
- Модель сессии трекинга (начало/конец поездки) для хранения завершенных маршрутов.
- UI-страница карты на Яндекс.Картах для записи/просмотра треков.
- Минимальные автоматизированные тесты и логи ошибок записи телеметрии.

## Риски и меры снижения
- Дрейф времени и неоднозначность таймзоны -> использовать стратегию UTC и документировать правила конвертации.
- Высокая нагрузка на запись -> держать сущность компактной, использовать корректные индексы и дефолтную пагинацию.
- Проблемы качества данных из внешних источников -> строгая валидация и понятные причины отклонения.
- Нестабильная связь/геолокация в браузере -> буферизация точек и повторная отправка при восстановлении сети.
- Риски интеграции с реальным GPS-трекером -> не включать в MVP защиты, оставить унифицированный API и описать будущий адаптер.
- Риски работы с Яндекс.Картами -> заранее добавить API-ключ в конфигурацию и подготовить fallback-сообщение при ошибке загрузки карты.
